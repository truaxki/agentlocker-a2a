AgentLocker × A2A — Hub Architecture

Personal agent mesh — Vercel dashboard + Supabase auth + A2A-compliant agents on VPS

What is an Agent Card?

  • • JSON file at /.well-known/agent-card.json
  • • Declares agent name, description, endpoint URL
  • • Lists skills with input/output modes
  • • Specifies auth scheme (bearer, OAuth)
  • • Declares streaming & push notification support
  • • AgentLocker discovers any agent that publishes one

Why A2A for AgentLocker?

  • • One standard — any agent can join the hub
  • • No custom integrations per agent
  • • Agents delegate tasks to each other natively
  • • Linux Foundation governed — no vendor lock-in
  • • Streaming built-in for long-running tasks
  • • 100+ enterprise partners already committed

Phased Rollout

  • • Phase 1: nginx + HTTPS on VPS, Hermes Agent Card live
  • • Phase 2: Supabase OAuth, Vercel login gate
  • • Phase 3: AgentLocker discovers & displays Agent Cards
  • • Phase 4: Task dispatch UI, SSE streaming results
  • • Phase 5: RSR onboarded, cross-agent delegation
